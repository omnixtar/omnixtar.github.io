### 案例：计算矩形面积与处理不同数据来源

假设我们需要完成一个简单的计算任务：**求矩形的面积（长 × 宽）**。

#### 1. 传统程序语言（以 Python 为例）的实现方式

在传统语言中，函数通过**命名变量**作为输入参数和返回值，与变量高度绑定。

```python
# 定义函数：明确绑定输入变量 length, width，以及返回值 area
def calculate_area(length, width):
    area = length * width
    return area

# 调用场景 1：直接传入数值
result1 = calculate_area(10, 20)

# 调用场景 2：从字典中取值传入
data = {"高度": 5, "宽度": 8}
result2 = calculate_area(data["高度"], data["宽度"])

# 调用场景 3：从数组取值传入
dimensions = [15, 25]
result3 = calculate_area(dimensions[0], dimensions[1])
```

**问题与绑定分析：**
- `calculate_area` 的函数签名（signature）强行规定了输入的**顺序**（第一个是长，第二个是宽）和**变量名**（length, width）。
- 如果数据源变了（例如字典里叫 `height` 和 `width`，数组里是 `[宽, 长]`），调用者必须手动提取变量并**严格按照形参顺序**传递。
- 如果我想复用这个逻辑去计算（长×长）或（宽×宽），必须重新编写调用代码或定义新函数。
- **核心痛点**：函数逻辑被“输入变量名称”和“输入变量顺序”所禁锢。

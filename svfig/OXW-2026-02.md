### Core Thesis
The presentation extends Bitcoin’s core idea — **hash of public key as user identifier** — into **Omnihash**, a universal hashcode system for representing **any** digital asset and its ownership. Combined with **Phoscript** (a FORTH variant), it forms **Omni*Web**: a truly decentralised web ecosystem owned and operated by individuals and free-software programmers.

The two flagship applications are:
1. **AGI self-identity**: Give AI agents a persistent cryptographic “self” via hash-of-public-key.
2. **Decentralised AI GPU token market**: Monetise idle GPU resources on personal computers and high-end NVIDIA H-class servers using Omnihash tokens (credit-based or cash/crypto settlement).

### Key Building Blocks
- **Omnihash**: Deterministic, base64-encoded hash (often SHA-256) used as universal ID for users, documents, actions (“like”, “share”), conversations, etc. Enables **DJSON** (Decentralised JSON) — self-verifying, ownership-encoded data structures that need no central registry.
- **UIPC (Unified Inter-Process Communications)**: I3P (iframe + I2P) + ngrok + Python Selenium. Middle layer that makes the stack homoiconic (code = data) and practical for FORTH programmers.
- **Omniscientia**: Open-source clone/extension of the popular “OpenClaw” AI shell. Functions as:
  - Fusion engine merging responses from all major AI services (Gemini, ChatGPT, Grok, Claude, etc.).
  - Omniscient search engine (AI services normally don’t share data).
  - Grok-style shared conversations extended to **all** AIs and **all** users via Omnihash.
- **Hash Filesystem (HFS)**: Directory names = hash-of-public-key (user ID). “Dual chained properties” of hashes replace much of traditional database functionality.

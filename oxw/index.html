I2P: A Brief Overview

I2P is essential a clever program to bypass the thought-to-be ubiquitous Domain Name System, and it is open source and written in Java programming language.

As such, there are plenty of junior and senior programmers who can verify the validity of the code, from its GitHub repository:

https://github.com/i2p/i2p.i2p

The installation JAR (Java Archive) file can be obtained from:

https://geti2p.net/en/

We shall skip the detailed steps which are explained comprehensively elsewhere, to provide a quick overview about I2P and Omni*Web to users and fellow programmers.

I2P Tunnel Manager

Once you completed the installation, you may start i2prouter with the following command:

~/i2p$ ./i2prouter start

Then open the following page:

http://127.0.0.1:7657/i2ptunnelmgr

Scroll down to the bottom half, you will see something similar to the screenshot above, where the following appear:

I2P HTTP Proxy 127.0.0.1 4444

I2P HTTPS Proxy 127.0.0.1 4445

Make sure they are running with status light in green or yellow. Otherwise, simply click the "Start" button on the right.

When the client tunnels are already running, the text on the button on the right will be "Stop".

A client tunnel is an interface for your browser (client) to connect to the web server, which has a corresponding "server tunnel", which we shall explain later.

Firefox Proxy Settings

Next we set up the "proxy" for the browser (Firefox) as shown above, using the parameters from I2P Tunnel Manager.

Once you do this, you will be able to connect to Omni*Web test server shown at the beginning of this tutorial, by clicking the big green "Get Started" button. (One at the start of this web page, one below here, both performing same function.)

We believe this is the easiest way to get new users to use I2P, as they do not have to remember the long I2P address, and will feel safe after reading and going through the steps shown above on github.io.

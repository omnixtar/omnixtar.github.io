## TMX 三类货币交易 DJSON 示例

如果说 DJSON (去中心化JSON) 是由 [Omnihash 全杂数](./contract) 及任意字串形成， 那么 DJSON 可以**表示任何形式的数字资产**,以及该数字资产的**所有权的转换的记录**。 


TMX (Trispecies Monetary Transaction) 三类货币交易: 以 DJSON 表示 -- 三类货币(fiat 法币、bullion 金银币、cryptocurrency 加密币)交易的记录。 

- 例如： GRATIS → BTC → ETH → GRATIS
  - 详见 [[三币交易扩展：三角套汇示例]](https://omnixtar.github.io/fcn/a3#%E4%B8%89%E5%B8%81%E4%BA%A4%E6%98%93%E6%89%A9%E5%B1%95%E4%B8%89%E8%A7%92%E5%A5%97%E6%B1%87%E7%A4%BA%E4%BE%8B)

在此基础上:   
- [**GRATIS代币**](../fcn/a1) (Grand Unified AI Token Trading Platform 统一AI代币交易平台): 构成激励层，用微支付奖励每一个贡献算力的参与者。
  - 拉丁语 GRATIS "免费": 隐喻 GRATIS代币**不需要用户付费**获得, 而是:
    - 通过 **Omnihash 全杂数** 实现 **以用户的数字产权收费, 换取使用 AI词元算力 的许可**。

## DJSON 格式说明

根据网页定义，DJSON 是一个 JSON 对象或其编码字符串，其中至少有一个字段是 Omnihash，代表该 JSON 对象的所有者。

网页中给出的 DJSON 示例格式为：

`["2025-02-11T14:25:28.207+0000","like","CXAGcRKevA==","CXAGcRKevA==","HymWBzfj9A==","HymWBzfj9A== s: x:"]`

字段说明：
- `timestamp`：时间戳
- `action`：操作类型（如 `like`、`comment`、`share` 等）
- `current_user_ID`：当前用户的 Omnihash 标识符
- `prev_msg_owner`：前一条消息所有者的 Omnihash
- `doc_hash`：文档的 Omnihash（URL 的哈希值）
- `messages`：消息内容

## TMX 三币交易 DJSON 示例

基于上述格式，TMX（Triple-Market Exchange，三币交易）的 DJSON 可设计如下：

```json
[
  "2026-07-14T10:30:00.000+0800",
  "tmx_trade",
  "CXAGcRKevA==",
  "HymWBzfj9A==",
  "DgV6_qnujw==",
  {
    "pair": "GRATIS/BTC",
    "side": "buy",
    "price": "0.00000042",
    "quantity": "10000",
    "total": "0.0042",
    "fee": "0.0000042",
    "fee_token": "GRATIS",
    "tx_hash": "3a4b5c6d7e8f9a0b1c2d3e4f5a6b7c8d9e0f1a2b3c4d5e6f7a8b9c0d1e2f3a4b5c6d",
    "expiry": "2026-07-14T11:00:00.000+0800"
  }
]
```

### 字段逐项解释

| 字段 | 值 | 说明 |
|------|-----|------|
| `timestamp` | `2026-07-14T10:30:00.000+0800` | 交易发起时间 |
| `action` | `tmx_trade` | 操作类型，表示三币市场交易 |
| `current_user_ID` | `CXAGcRKevA==` | 当前交易发起用户的 Omnihash 标识符 |
| `prev_msg_owner` | `HymWBzfj9A==` | 前一笔交易或对手方的 Omnihash |
| `doc_hash` | `DgV6_qnujw==` | 交易合约或市场文档的 Omnihash |
| `messages` | 嵌套 JSON 对象 | 交易详情（见下表） |

### `messages` 字段详解

| 字段 | 值 | 说明 |
|------|-----|------|
| `pair` | `GRATIS/BTC` | 交易对：GRATIS 代币对比特币 |
| `side` | `buy` | 交易方向：买入（或 `sell` 卖出） |
| `price` | `0.00000042` | 单价：每单位 GRATIS 的 BTC 价格 |
| `quantity` | `10000` | 数量：交易的 GRATIS 数量 |
| `total` | `0.0042` | 总价：`price × quantity` |
| `fee` | `0.0000042` | 手续费 |
| `fee_token` | `GRATIS` | 手续费支付币种 |
| `tx_hash` | `3a4b...` | 交易哈希（区块链交易 ID） |
| `expiry` | `2026-07-14T11:00:00.000+0800` | 交易过期时间 |

## 三币交易扩展：三角套汇示例

若 TMX 指代三种货币之间的三角交易（如 GRATIS → BTC → ETH → GRATIS），则可扩展为：

```json
[
  "2026-07-14T10:30:00.000+0800",
  "tmx_triangle",
  "CXAGcRKevA==",
  "HymWBzfj9A==",
  "DgV6_qnujw==",
  {
    "legs": [
      {"from": "GRATIS", "to": "BTC", "amount": "10000", "rate": "0.00000042"},
      {"from": "BTC", "to": "ETH", "amount": "0.0042", "rate": "15.5"},
      {"from": "ETH", "to": "GRATIS", "amount": "0.0651", "rate": "153846.15"}
    ],
    "arbitrage_profit": "0.0001",
    "profit_pct": "0.0001",
    "tx_hashes": ["0x3a4b...", "0x5c6d...", "0x7e8f..."],
    "expiry": "2026-07-14T10:35:00.000+0800"
  }
]
```

## DJSON 的核心理念

DJSON 通过将 Omnihash 嵌入 JSON 字符串，实现了：

1. **去中心化身份**：用户标识符基于公钥哈希，无需中心化注册
2. **类型保持**：哈希值可参与运算并产生新的哈希，支持复杂操作组合
3. **所有权证明**：Omnihash 代表数字资产的所有者，可用于版权、交易和合约

这种设计使得 DJSON 能够承载从社交媒体互动（点赞、评论、分享）到金融交易（支付、借贷、投资）等各种去中心化应用场景。

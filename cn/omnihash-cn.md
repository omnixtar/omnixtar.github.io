## Omnihash 与 DJSON 去中心化 JSON (定义)

**Omnihash 是：**

1. **（基础形态）** 用户公钥的哈希值（hash of user's public key, HPBK）的 Base64 编码字符串，用于表示用户身份；

1. **（复合形态）** 一个 DJSON 的哈希值的 Base64 编码字符串，该 DJSON 至少包含一位用户的 HPBK（用户 A 的 HPBK-A），用以表示该 DJSON（代表某件数字资产）归属于该用户。

**DJSON（去中心化 JSON）** 是一个 JSON 对象或其编码字符串，其中至少有一个字段是 Omnihash，用以表示该 JSON 对象的所有者。

**Omnihash 示例：** `DgV6_qnujw==`

DJSON 或去中心化 JSON 是 Omni\*Web 的一项关键突破——代表各类数字资产与实体（从用户标识符到社交媒体动作，如点赞、评论、分享）的 Base64 哈希码，被嵌入到看似普通的 JSON 字符串中。

底层去中心化 JSON 是比特币地址的延伸——比特币地址源于公钥哈希，用作用户标识符。将公钥哈希泛化为用户标识符，是去中心化计算领域的一项突破，因为此前基于区块链或加密货币的框架严重受制于矿工的垄断。

令 DJSON 如此特别而强大的，是我们所称的“哈希数与整数的类型保持性质”——这源于整数的环论性质，即整数上的加法与乘法运算其结果始终为整数。

### 如何为这篇 GitHub Markdown 文章添加点赞功能

我们先公布答案，再作解释——我们假设有些读者已经等不及了：


生成本文“点赞”DJSON 的步骤如下（附视频演示）：

1. 将此网页的副本保存为 Omni\*Web 服务器上的本地文件。
2. 为此文档生成 `doc_hash`。
3. 以 `doc_hash` 为名为此文档创建子目录。
4. 复制必要文件与软链接。
5. 在 [Omni\*Web 服务器](https://yo6sgmfq7pfvvp2e4kcuhjtfg7wfltt63igwcukhbmuqm6lu3a3a.b32.i2p/h/E5bnwoBdvg==/?nn=z) 上通过 I2P（隐形互联网项目）及 `doc_hash` 打开本地副本。

7. 用用户的公钥刷新身份验证令牌。

# Omni*World

## Resources

**Here is a curated list of existing real-world resources, projects, websites, and tools** that align with the **Omni*World** vision (Three.js-based real-world simulation from video, intelligent actors like cars, decentralized protocols, Lisp-inspired logic, video-to-3D reconstruction, and EV/world modeling).

### 1. Core Simulation & 3D Engine
- **Three.js** — The foundational JavaScript 3D library for WebGL/WebGPU rendering. Official site with hundreds of examples (including video textures, physics integrations).  
  → https://threejs.org/ | https://github.com/mrdoob/three.js
- **Three.js Examples** — Video textures, panoramas, post-processing, and more.  
  → https://threejs.org/examples/
- **React Three Fiber** — Declarative Three.js in React (popular for complex scenes).  
- **Three.js Car Physics Projects**:
  - DJ-Laser/threejs-car (Rapier physics)
  - Various Cannon.js / Rapier vehicle demos on Three.js Discourse
  - Straight Car tutorial (sbcode.net)

### 2. Omnihash & Decentralized Protocol (Fiction-Inspired Real Roots)
- **omnixtar.github.io/contract** — The core reference for Omnihash (hashing for any digital asset: photos, videos, simulations, ownership/provenance).  
  → https://omnixtar.github.io/contract/
- **Omnihash GitHub org** — Related modules and contracts.  
  → https://github.com/omnihash

### 3. Video-to-3D / World Model Reconstruction
- **Awesome-From-Video-Generation-to-World-Model** — Curated list of papers/projects turning video into generative world models.  
  → https://github.com/ziqihuangg/Awesome-From-Video-Generation-to-World-Model
- **OpenAI Sora-era papers** — “Video generation models as world simulators” (foundational thinking).  
- **TripoSR** — Fast open-source 2D-to-3D reconstruction.  
- **NeuralRecon, DROID-SLAM, VisFusion** — Real-time 3D scene reconstruction from monocular video.
- **Vid2World** — Video diffusion → interactive world models.

### 4. Decentralized Compute & AI Networks (for sharded simulation)
- **io.net** — Decentralized GPU marketplace for AI workloads.
- **Akash Network (AKT)** — Decentralized cloud/GPU compute.
- **Render Network (RNDR)** — Decentralized GPU rendering & AI inference.
- **Bittensor (TAO)** — Decentralized machine learning / intelligence network.
- **Other DePIN projects** (Aethir, Gensyn, etc.) — Distributed compute for 2026+ scale.

### 5. Lisp / Symbolic AI in JavaScript
- **LIPS Scheme** — Powerful Scheme (Lisp dialect) interpreter in JavaScript, R7RS compliant, mixes with JS.  
  → https://lips.js.org/
- **Kraks/lisp.js** — Tiny Lisp interpreter in JS.
- **Other Lisp-to-JS** — Compilers and AI-agent Lisp projects (e.g., PTC-Lisp, Sema).

### 6. Physics & Intelligent Actors (Cars / Vehicles)
- **Rapier** / **Cannon.js** / **Ammo.js** — Physics engines integrated with Three.js for realistic car simulation.
- **Physijs** — Older but direct Three.js physics plugin.
- Open-source car controllers and vehicle sims on GitHub (e.g., abhicominin/Car-Controller-Threejs).

### 7. Datasets & Real-World Video Sources
- **EV / Autonomous Driving Datasets**: Argoverse, BDD100K, nuScenes, Waymo Open Dataset (real camera data from cars).
- **YouTube + Public Video** — Unlimited source for dashcam, traffic, urban footage (as highlighted in the fiction).
- **WildWorld, MultiScene360**, and other multi-view/action-conditioned datasets.

### 8. Other Useful Tools & Ecosystems
- **GLTF loaders & 3D model repos** (Sketchfab, etc.) for importing realistic cars/actors.
- **TensorFlow.js / ONNX** — Run ML models (perception, agents) directly in browser.
- **Post-processing libraries** for Three.js (bloom, DOF, SSR for realism).
- **Three.js community**: Discourse forum, GitHub showcases, Codrops tutorials (e.g., interactive video projection mapping).

These resources form a **practical foundation** to start building Omni*World today — combining Three.js visuals, video reconstruction pipelines, decentralized compute via DePIN networks, Lisp-style symbolic layers, and public video data to counter proprietary EV fleets (Tesla, BYD, etc.).
